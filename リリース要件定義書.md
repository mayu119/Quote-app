## 1. ターゲット（1行）

> **「自分を変えたい」と思いながら毎朝なんとなくスマホを開いてしまう20-35歳の男性**

- 「変えたいけど行動できてない」男が対象
- すでにストイックに動いてる層は対象外（コンテンツを消費しない）
- Alpha / Sigma Male 文化に共感する層
- 筋トレ・自己啓発・ビジネスに興味はあるが「始めてない」or「続いてない」

## 2. 競合分析 — 低評価レビューから抽出した5つの致命的弱点

> **調査日**: 2026-02-20
> **調査対象**: iOS日本ストアの名言/モチベ系アプリ8本
> **方針**: 競合の★1レビューで繰り返し出る不満 = ユーザーが本当に求めてること。これを潰せば勝てる。

### 調査対象アプリ

| アプリ名 | 評価 | レビュー数 |
|---|---|---|
| Motivation - 今日の名言 | 4.68 | 3,412 |
| Motto - 名言・自己肯定感・モチベーション | 4.71 | 7,601 |
| 名言まとめ | 4.37 | 856 |
| 名言ウィジェット | 4.52 | 83 |
| エミネンス モチベーションを上げる | 4.72 | 367 |
| 名言メーカー | 5.0 | 1 |
| 名言＆壁紙 | 5.0 | 1 |
| 毎日自己モチベーション名言… | 4.81 | 21 |

### 弱点①：通知品質 — 「来ない」「同じ文言ばかり」

| 競合の実態 | うちの対策 |
|---|---|
| Motto:「通知が同じ内容」 | 重複排除ロジック必須。一定期間内に同じ名言が出ない制御 |
| 名言まとめ:「通知が来ない」 | ローカル通知で確実に配信。サーバー依存しない |
| Motivation:「ウィジェットが同じ名言ばかり」 | ウィジェットも1日1回確実に更新 |

→ **対応**: 機能①「毎日1名言の自動プッシュ通知」で直接潰す

---

### 弱点②：表示バグ — コア体験を壊す不具合

| 競合の実態 | うちの対策 |
|---|---|
| 名言まとめ:「文字が表示されない（空欄）」 | 名言データのバリデーション必須。空文字チェック |
| 全体:「表示崩れ」「スタックする」 | SwiftUIでレイアウト崩れを根本排除。長文対応テスト |
| 全体:「文字が空白」 | フォールバック名言を必ず用意（データ欠損時にも表示） |

→ **対応**: リリース判定条件⑥「クラッシュなし、文字化け/空白表示なし」で死守

> ⚠️ **これは競合の最大弱点。ここで★1が発生してる。品質だけで差別化できる領域。**

---

### 弱点③：課金圧 —「課金しないと実質使えない」

| 競合の実態 | うちの対策 |
|---|---|
| Motivation:「課金必須感が強い」 | 無料でも基本機能（1日1名言・お気に入り・シェア）は使える |
| Motto:「課金しないと機能制限が強い」 | 「使えない」ではなく「もっと欲しくなる」設計 |
| 全体: 即★1の原因 | 無料体験で価値を感じさせてから課金に導く |

→ **対応**: マネタイズ設計参照。課金自体は妥当。問題は「無料で何もできない」こと。

> **設計原則**: 無料で基本体験は成立させる。ただし課金すること自体は正しい——価値あるものに金を払うのは当然。「課金したくなる」設計が正解。

---

### 弱点④：広告体験 — 短時間利用アプリとの相性最悪

| 競合の実態 | うちの対策 |
|---|---|
| 名言まとめ:「広告が重い/多い」 | **広告を入れない。完全にゼロ。** |
| 全体: 動画広告が体験を切る | この世界観に広告は異物。入れた瞬間に安っぽくなる |
| 全体: 短時間利用アプリで広告は致命的 | 広告収益なんてたかが知れてる。サブスク一本で潔く |

→ **対応**: 広告なし。収益はサブスクのみ。

> **判断根拠**: 「オスになれ」と煽っておいて画面にゲーム広告が出たら台無し。広告ゼロであること自体がこのアプリの世界観の一部。

---

### 弱点⑤：再訪導線の不足 — お気に入り/検索がない

| 競合の実態 | うちの対策 |
|---|---|
| Motto:「検索機能がない」 | v1ではお気に入り一覧でカバー。検索はv2 |
| 名言ウィジェット:「お気に入り登録まわりの不具合」 | お気に入りのUXを最優先でテスト |
| 全体:「いい名言を再訪できない」 | ワンタップ保存 + 一覧画面を必須機能に含める |

→ **対応**: 機能③「お気に入り保存」で直接潰す

---

### 競合分析サマリ（設計原則として）

```
競合が負けてる理由:
 1. 通知が来ない/被る    → うちは確実配信 + 重複排除
 2. 表示バグで★1量産     → うちは品質死守（これだけで勝てる）
 3. 課金しないと使えない  → うちは無料で基本体験を成立させた上で課金に導く
 4. 広告で体験ぶつ切り    → うちは広告ゼロ。世界観を壊さない
 5. いい名言に再会できない → うちはお気に入り + シェアで解決
```

> **勝ち筋**: 競合は「機能は多いが品質が低い」。うちは「機能は3つだが品質が圧倒的に高い」で出る。

## 3. 必須機能（リリースに含む3つ）

### ① 毎日1名言の自動プッシュ通知 + メイン画面表示

**なぜ必須か**: コアの行動嗜癖トリガー。競合の「通知来ない/同じ文言ばかり」を潰す。

| 仕様項目 | 内容 |
|---|---|
| 通知タイミング | ユーザーが設定可能（デフォルト: 朝7:00） |
| 通知文言 | 釣り場理論準拠。名言そのものを出さず、開かせる文言 |
| メイン画面 | 名言全文 + 偉人名 + 高品質背景画像 |
| 名言選定ロジック | ランダム（同じ名言が期間内に被らない制御付き） |
| 無料ユーザー | 1日1名言のみ閲覧可 |
| プレミアム | 過去の名言も含め無制限閲覧 |

**通知文言の設計（釣り場理論）**:
- ❌「今日の名言：人生を変えたいなら行動しろ — ○○」（答え完結、開く理由がない）
- ✅「今日の名言、読んだ瞬間に鳥肌立った...」
- ✅「昨日と今日の名言を比べてみろ。気づくことがある。」
- ✅「90%の男がスルーする。でもお前は違うと信じてる。」

---

### ② ホーム画面ウィジェット

**なぜ必須か**: ロック解除のたびに名言が目に入る = 接触回数最大化。このアプリの存在意義の半分。

| 仕様項目 | 内容 |
|---|---|
| サイズ | Small（2×2）/ Medium（4×2）の2種 |
| 表示内容 | 短縮版パンチライン（1-2行） + 偉人名 |
| タップ時 | アプリ起動 → 全文 + 背景画像 + 解説表示 |
| 更新頻度 | 1日1回（名言更新と連動） |
| デザイン | 暗めトーン、ミニマル、テキスト中心 |
| 対応OS | iOS 17+ |

**二層設計**:
- **ウィジェット**: 短縮版（刺さるフレーズだけ）→「続きが気になる」
- **アプリ本体**: 全文 + 背景画像 + 偉人の文脈解説

---

### ③ お気に入り保存 + SNSシェア（共有画像生成）

**なぜ必須か**: お気に入り = 「自分だけのコレクション」→ 定着。シェア = バイラルエンジン。

| 仕様項目 | 内容 |
|---|---|
| お気に入り | ワンタップで保存。一覧画面あり |
| シェア画像生成 | 名言 + 高品質背景 + アプリロゴ透かし |
| シェア先 | Instagram Stories / X / LINE / 汎用シェアシート |
| シェアデフォルト文言 | 釣り場理論準拠（下記） |
| 無料/プレミアム | お気に入り保存は無料。無制限 |

**シェア文言の設計（Stronger方式）**:
- ❌「いい名言を見つけました！ #名言アプリ」
- ✅「これ読んで朝から○○する気になった。読まない方がいいかも。」
- ✅「毎朝これ読んでるけど、今日のはちょっとヤバかった。」

**シェア画像クオリティ基準**:
- Instagram Storiesに投稿しても「ダサい」と思われないレベル
- 暗めシネマティック背景 + 明るいテキスト + ロゴ透かし
- 「この画像カッコいいからシェアしたい」と思わせる

## 4. やらない機能（リリーススコープ外）

| 機能 | なぜ今じゃないか | いつやるか |
|---|---|---|
| 「オスレベル」ストリークシステム | コア体験（読む→響く→シェア）が先 | v2: リテンションデータを見てから |
| カテゴリ検索・偉人検索 | DB量が少ないうちは検索の意味がない | v2: 名言500件超えたら |
| 瞑想モード | ユーザーが何を求めてるか分かってから | v3以降 |
| AI個人化名言セレクション | 技術的に重い & データがないと最適化不可 | v3以降 |
| Ultimateプラン（年4,800円） | まずプレミアム1本で検証 | 課金率データを見てから |
| ソーシャル機能（いいね・コメント） | UGCはSNSでやればいい | 不要の可能性高い |

## 5. リリース判定条件（全て満たしてリリース）

### ✅ MUST（1つでも欠けたらリリースしない）

| # | 条件 | 基準 |
|---|---|---|
| 1 | **コア体験の完走** | 通知タップ → 名言表示 → お気に入り or シェア、が途切れずに動く |
| 2 | **名言データ 200件以上** | 毎日1件で約7ヶ月分。被らない体験を保証 |
| 3 | **ウィジェット安定動作** | iOS 17+で表示崩れ・更新停止なし |
| 4 | **シェア画像品質** | Instagram Storiesに出して恥ずかしくないクオリティ |
| 5 | **課金実装完了** | 無料（1日1名言・広告なし）+ プレミアム（月480円 / 年2,900円）がRevenueCatで動作 |
| 6 | **品質基準** | クラッシュなし、文字化け/空白表示なし（競合の最大弱点を絶対に踏まない） |

### ❌ NGライン（これに該当したら絶対にリリースしない）

| 条件 | 理由 |
|---|---|
| 名言が100件未満 | 2-3ヶ月で被る → 体験として破綻 |
| ウィジェットが不安定 | このアプリの存在意義が半減 |
| シェア画像がダサい | バイラルしない = マーケ戦略が機能しない |
| 通知が来ない/遅延する | 毎朝の習慣化が成立しない |

## 6. 技術スタック（想定）

| 項目 | 選定 |
|---|---|
| プラットフォーム | iOS（Swift / SwiftUI） |
| 最低対応OS | iOS 17+ |
| 課金基盤 | RevenueCat |

| 名言DB | JSON or ローカルDB（サーバー不要） |

| プッシュ通知 | ローカル通知（サーバー不要） |
| 画像生成（シェア） | Core Graphics / UIKit renderering |

> 初期はサーバーコストゼロで運用し、ユーザー数に応じてサーバー移行を検討。

## 7. マネタイズ設計

> **方針**: 広告なし。サブスク一本。「オスになれ」という世界観に広告は異物。広告ゼロ自体がブランド。

| プラン | 内容 | 価格 |
|---|---|---|
| **無料** | 1日1名言 / ウィジェット（短縮版）/ お気に入り / シェア / 広告なし | ¥0 |
| **プレミアム** | 無制限閲覧 / フル背景画像 / 過去名言アーカイブ / プレミアム限定名言 | 月480円 / 年2,900円 |

**なぜ広告なし?**:
- この世界観に広告を入れた瞬間に安っぽくなる
- 短時間利用アプリの広告収益はたかが知れてる
- 広告ゼロ = 「このアプリは本気だ」というメッセージになる
- 競合が広告まみれでユーザーが嫌がってるのに、同じことをする理由がない

**課金導線**:
- 「昨日の名言をもう一度見たい」→ プレミアムへ
- 「もっと名言を読みたい」→ プレミアムへ
- 「プレミアム限定の名言が気になる」→ プレミアムへ
- ウィジェットの全文表示 → プレミアムへ（v2検討）

## 8. デザイン方針

| 項目 | 方針 |
|---|---|
| トーンカラー | ダーク基調（黒/ダークグレー） + アクセントカラー（ゴールド or 赤） |
| フォント | 力強さのあるサンセリフ（日本語：ヒラギノ角ゴ / 英語：SF Pro Display Bold） |
| 背景画像 | シネマティック系（都市夜景 / ジム / 荒野 / 格闘技）— 名言のジャンルに合わせる |
| UI密度 | 極ミニマル。名言以外の要素を極力排除。「開いて3秒で名言が目に入る」 |
| アニメーション | テキストフェードイン / 背景の微かなパララックス |
